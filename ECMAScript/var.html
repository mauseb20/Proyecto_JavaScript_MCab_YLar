var vs let vs const

var

Descripción

var es la forma más común de declaración de una variable en Javascript. Una vez que se crea una variable como var, esta tendrá un ámbito global, por lo tanto, se podrá actualizar o redefinir en cualquier momento.


    var width = 100;
    console.log(width); // 100
    width = 200;
    console.log(width); // 200

Caracteristicas

var se la puede definir dentro de una función y usarla.


    function setWidth() {
      var width = 100;
      console.log(width);
    }
    setWidth();

Pero si tratamos de usarla fuera de la función que fue definida, esta no se imprimirá y sale un error de que no ha sido declarada.


    function setWidth() {
      var width = 100;
      console.log(width);
    }
    setWidth();
    console.log(width); // error, width no esta definido

Esto se debe a que dentro de la función es declarada como local y solo en estos casos es que se la define como local.

let

Descripción

Como var permite redefinirla, se daba los problemas de que se podía declarar dos variables con el mismo nombre dentro de una misma pantalla. Para evitar este inconveniente se tiene las variables que se declaran con let.

Si se tiene la declaración de las siguientes variables:


    let points = 50;
    let winner = false;

si se desea re-declarar la variable let:


    let points = 60;

El browser lanzará un error indicando que dicha variable ya ha sido definida. Esta variable se puede actualizar, pero no se permitirá declarar dos veces con el mismo nombre.

Caracteristicas

Si se tiene el siguiente código:


    let points = 50;
    let winner = false;

    if(points > 40) {
       let winner = true
    }
    Console.log(winner); //false

El resultado es false, esto se da porque let declara la variable dentro de su propio ámbito, este tipo de variables no son hoisted. En este caso se tienen dos variables diferentes, aunque tengan el mismo nombre, por estar en diferentes ámbitos.


    let winner = false //fuera del lazo if, está delimitada al uso en la ventana.
    let winner = true //dentro del lazo if, está delimitada al uso exclusivo del bloque.

const

Descripción

Otro tipo de variables usadas son las const, estas variables tienen la misma definición que las variables let, la diferencia entre estas dos, es que las variables let, se la puede actualizar, pero las variables const, van a tener un solo valor, no se las puede actualizar.

Si se tiene el siguiente código:


    const key = 'abc123';
    let points = 50;
    let winner = false;

    points = 60;

Este va a funcionar correctamente. La variable points se actualizará sin ningún problema.

Pero con el siquiente codigo:


    const key = 'abc123';
    let points = 50;
    let winner = false;

    key = abc1234;

Este no va a funcionar, porque no se puede actualizar una variable const.

Caracteristicas

Se puede definir un objeto completo con tipo const:


    const person = {
       name: 'Wes',
       age: 28
    }

El objeto en sí es inmutable. Por ejemplo, si se desea redefinir el objeto tipeando person = { name: ‘Wesley’}, esto sin duda dará un error, ya que no se puede redefinir al objeto completo.

Pero si se pueden actualizar sus propiedades individualmente, como se lo muestra en el código a continuación:


    const person = {
       name: 'Wes',
       age: 28
    }

    person.age = 29

Esto no dará problema y correrá sin ninguna dificultad.
